# 跨任务复盘库 (Lessons Learned)

> Phase 1 Discovery 自动按"模块 + 任务类型"召回本文件命中条目,写入新任务文档"第 2 节 历史相似坑"小节。
> Phase 7 Summary 按"沉淀判定四问"判定本任务踩坑是否值得提炼,值得才追加(每任务**最多 1 条**)。
> 同一条触发次数 ≥3 → 移到顶部"高频坑"区(任何任务都看)。
>
> **条目格式**:`- <祈使句一句话规则>(来源: <task 文件>;触发次数: N)`
>
> **沉淀判定四问**(全部 yes 才追加,任一 no 跳过):
> 1. **跨任务共性?** — 同类问题在别的任务里也会出现?(项目特异 quirk → 写 docs/,不进 lessons)
> 2. **可重现?** — 能预期下次某场景必触发?(纯偶发 / 一次性 → 不沉淀)
> 3. **可操作?** — 写成"做 X / 不做 X"的祈使句?(只能描述现象 → 不沉淀)
> 4. **非已有?** — 库中无同类?(已有 → 仅触发次数 +1)
>
> **反例**(不沉淀):
> - ❌ "本项目 webpack 5 升级后 import.meta 报错"(项目特异,放 docs/stack/)
> - ❌ "本周三 CI 偶发超时"(偶发)
> - ❌ "团队代码风格不一致"(无操作指令)
> - ❌ "测试覆盖率重要"(空话,不可操作)
>
> **正例**:
> - ✅ "ADR §影响/改动范围必须列调用入口文件,不只列实现文件"(跨任务 / 可重现 / 可操作)
> - ✅ "改 DB schema 字段名前先 grep 字段名 in {code_roots}/ 防漏改 caller"(同上)

## 召回索引说明

Phase 1 Discovery 召回时按以下顺序查:

1. 「高频坑」段(无条件读全部)
2. 「模块:<当前任务模块>」段(精确匹配,如 `feature/auth` `stack/backend`)
3. 「跨模块通用」段(无条件读全部)

**实现伪代码**(供 Claude 主流程参考):

```bash
# 1. 高频坑段(从 "## 高频坑" 到下一个 "## " 二级标题)
awk '/^## 高频坑/,/^## (?!高频坑)/' {lessons_path}

# 2. 当前模块段
awk -v m="$current_module" '
  $0 ~ "^## 模块:"m"$" {flag=1; next}
  /^## / && flag {flag=0}
  flag {print}
' {lessons_path}

# 3. 跨模块通用段
awk '/^## 跨模块通用/,/^## (?!跨模块通用)/' {lessons_path}
```

命中后输出格式:

```
Phase 1 历史坑提示(本任务命中 N 条):
  1. <规则>(来源 <task>)
  2. <规则>(来源 <task>)
  ...
本任务我会重点对照执行。
```

同时把命中条目复制写入新任务文档"第 2 节 根因分析"下的"历史相似坑"小节,作为永久留痕。
